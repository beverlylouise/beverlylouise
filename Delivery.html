Scheduling Application

Python CSV data structures algorithms Visual Studios
Github Repository

Summary and notes: There are three trucks, two drivers, and 40 packages to deliver in Salt Lake City, Utah. The total combined mileage for all trucks needs to be under 140 miles. Each truck can hold 16 packages and the drivers can leave as early as 8 am. There are special notes on packages and one package has the wrong address but will be updated in the delivery system at 10:20 am. I am provided a map and a chart with distances. The trucks and packages need to be tracked and their status updated. This was a fun project to code. I love puzzles and the program was not too hard to write. I chose to use the “Nearest Neighbor” algorithm. I put my time and mileage tracker into the algorithm to process the delivery and status updates at the same time. I learned how to put a CSV file into a program. I also learned how to utilize enumerate in python. As it was not required, I did not design the program to solve the best way to deliver the packages, as that did not change. I figured out an efficient way to load the packages on trucks and design a route. If this was an ongoing delivery problem with new packages and addresses daily, I would have designed the program to load the truck on its own because that sounds like a fun challenge.

Tech Specs: This is a python application, coded in Visual Studios, that uses algorithms and data structures to design an efficient delivery route, meeting all delivery constraints while keeping the miles to a minimum.
